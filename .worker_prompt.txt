## Fix plan

### 1. Lenis tuning (make scroll feel native, not floaty)

`landing/src/lib/useLenis.ts` (or wherever Lenis is initialized):
- `lerp: 0.1` (was 0.08) — slightly snappier
- `duration: 0.9` (was 1.2) — closer to native momentum
- `wheelMultiplier: 1` — don't amplify scroll
- `smoothTouch: false` — let mobile use native (it's faster + matches platform)
- `easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10 * t))` — snappier exponential ease
- Disable Lenis if `window.innerWidth < 768` OR `prefers-reduced-motion: reduce`
- Make sure it's destroyed on unmount + only mounted once

### 2. Framer Motion durations (cut everything by ~40%)

Audit every `transition={{ duration: X }}` in `landing/src/components/**`. Defaults should be:
- Fades / opacity-only: `duration: 0.4, ease: [0.16, 1, 0.3, 1]` (out-expo)
- Translates / slides: `duration: 0.5, ease: [0.16, 1, 0.3, 1]`
- Hover micro-interactions: `duration: 0.18`
- Word-mask reveals (SplitText.tsx): `duration: 0.5`, stagger `0.03`
- Hero giant text reveal: `duration: 0.6` max

Anything currently `duration: 1.0+` → cut to `0.5-0.6`. Anything currently `0.6-0.9` → cut to `0.3-0.4`. Spring presets like `stiffness: 100, damping: 20` should be tightened to `stiffness: 200, damping: 25` for crisper response.

### 3. Scroll ↔ animation sync (the big one)

For each pinned StoryModule (IntakeMock, HuntMock, QualifyMock, SendMock):

- **Single source of truth:** one `useScroll({ target: sectionRef, offset: ['start start', 'end end'] })` per module. All sub-animations derive from this `scrollYProgress`.
- **Smooth the source:** wrap with `useSpring(scrollYProgress, { stiffness: 100, damping: 30, restDelta: 0.001 })` to remove ticker jitter without lagging.
- **Stepped useTransform:** for module phases, use stepped input arrays — e.g. typewriter typing in IntakeMock should be:
  ```ts
  const charProgress = useTransform(smoothProgress, [0.1, 0.6], [0, 1])
  ```
  Not `[0, 1] → [0, 1]` (mushy).
- **No competing scroll listeners.** If StoryModule.tsx has its own `useScroll`, the inner mock must NOT have a second one — pass `scrollYProgress` down as a prop.
- **Use `useMotionValueEvent` for stepped state changes** (e.g. "qualify accepted" check appearing) so the trigger fires once at the right scroll depth, not on every frame.

### 4. Custom cursor / scroll progress (perf)

`CustomCursor.tsx`:
- Use `useMotionValue` + `transform: translate3d(...)` directly. No React state in the hot path.
- Spring should be `stiffness: 350, damping: 35, mass: 0.5` — snappy, not floaty.
- Hide on touch via `(pointer: coarse)` media query — don't even mount the listener.
- `requestAnimationFrame` only if NOT using motion values directly.

`ScrollProgress.tsx`:
- Use `useScroll({ container: undefined })` once.
- The orange bar should be a single `motion.div` with `style={{ scaleX: scrollYProgress }}` and `transformOrigin: 0` — no useTransform, no extra subscriber.

### 5. Grain overlay (often the culprit)

`GrainOverlay.tsx`:
- Make sure it's `position: fixed; pointer-events: none; z-index: 60; mix-blend-mode: overlay; opacity: 0.04` and the SVG is rendered ONCE (memoize the data URL).
- Add `transform: translateZ(0); will-change: transform` to hint GPU compositing.
- If it's animated (subtle drift), animate `transform: translate3d(...)` — never `background-position`.

### 6. Section spacing rhythm

Define a single rhythm token in `landing/src/index.css` `:root`:
```css
--section-y: clamp(80px, 9vw, 144px);
--section-y-tight: clamp(56px, 6vw, 96px);
```

Then apply to every section in App.tsx / individual sections:
- Default vertical padding: `padding-block: var(--section-y)`
- Tight pairs (e.g. trust marquee following hero, FAQ following pricing): `padding-block: var(--section-y-tight)`
- Story modules already pin via height — leave their internal pin height alone, but ensure the wrapper around the 4 modules uses the rhythm token for the gap to next section.
- Footer gets only top spacing.

Also enforce horizontal: `padding-inline: clamp(20px, 4vw, 80px)` consistently — no rogue per-section margins.

### 7. GPU composition hints

Add `will-change: transform` (only) to:
- Pinned module wrappers
- Hero word-reveal spans
- Drag gallery track
- Custom cursor
- Curtain transitions

Don't add `will-change` everywhere — it backfires. Just the elements that move every frame.
